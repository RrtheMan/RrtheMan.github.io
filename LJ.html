Introduction

Throughout this project, I explored the fundamentals of secure web development, including password handling, form validation, and access control. While I successfully implemented core features such as encrypted password storage and admin-level access, I encountered challenges with certain security components like password recovery and password strength validation.

Despite time constraints, I applied essential countermeasures such as SQL injection prevention, XSS protection, and file upload validation. This project improved my understanding of real-world vulnerabilities and reinforced the importance of defensive coding practices. User Registration

Users can register an account by providing a username, password, and contact information. The form input is inserted into a MySQL database.Security Measures. Password Encryption: Passwords are hashed using a secure algorithm before insertion into the database. SQL Injection Protection: All user inputs are sanitized using mysqli_real_escape_string() to avoid malicious injection.

Secure Login

The login system verifies users by matching their input against the database records. Security Measures:Input fields are sanitized with mysqli_real_escape_string(). Passwords are compared using the hash stored in the database. Sessions are used to manage logged-in users.

Evaluation Request Page

Users can submit a request for item evaluation, including contact info, a comment, and a photo. Security Measures: XSS Protection: User comments are passed through htmlspecialchars() to prevent script injection. Input validation ensures only valid and safe text is stored in the database. Images are filtered by type and size.
